# Serralheria ERP (local-first)ERP para serralheria com foco em uso offline para vendedores e sincronização na loja.## Stack- **Frontend:** React, TypeScript, Vite, MUI, React Router, TanStack Query, React Hook Form, Zod, Recharts, Dexie, PWA (Workbox)- **Backend:** NestJS, TypeScript, Prisma, PostgreSQL, JWT- **Monorepo:** pnpm workspaces (`apps/web`, `apps/api`, `packages/shared`)## Pré-requisitos- Node.js >= 20- pnpm >= 9- Docker e Docker Compose (para PostgreSQL)## Rodar localmente### 1. Clonar e instalar```bashcd serralheria-erp-localfirstpnpm install```### 2. Variáveis de ambiente```bashcp .env.example .env```Ajuste `DATABASE_URL` se necessário. Padrão: `postgresql://postgres:postgres@localhost:5432/serralheria_erp`.### 3. Subir o banco (Docker)```bashpnpm docker:up```### 4. Migrar e seed (API)```bashcd apps/apipnpm prisma:generatepnpm prisma:migratepnpm prisma:seedcd ../..```### 5. Iniciar API e WebTerminal 1 (API):```bashpnpm dev:api```Terminal 2 (Web):```bashpnpm dev:web```- Web: http://localhost:5173- API: http://localhost:3001### Login demo- **Admin:** admin@serralheria.local / admin123- **Vendedor:** vendedor@serralheria.local / vendedor123## Scripts (raiz)| Script           | Descrição                ||------------------|--------------------------|| `pnpm docker:up`   | Sobe PostgreSQL (e pgAdmin) || `pnpm docker:down` | Para containers             || `pnpm dev:web`     | Sobe Vite (frontend)        || `pnpm dev:api`     | Sobe NestJS (API)           || `pnpm prisma:migrate` | Cria/aplica migrações (no api) || `pnpm prisma:seed`     | Roda seed (no api)            |## Estrutura```apps/web     → React + Vite + MUI + Dexieapps/api      → NestJS + Prismapackages/shared → Tipos + schemas Zod```## Próximos passos- [ ] Persistir dados no Dexie ao criar/editar Cliente, Obra, Orçamento e marcar `pendingSync`- [ ] Chamar `/sync` ao clicar em "Sincronizar" e aplicar resposta no Dexie- [ ] Service Worker com Workbox para PWA e cache- [ ] CRUD completo de Custom Fields e renderização dinâmica nos formulários- [ ] Workflow de status do Orçamento com transições por role- [ ] Editor de templates (proposta/contrato/OP)- [ ] Módulos Produção, Estoque, Financeiro, Relatórios (placeholders)
